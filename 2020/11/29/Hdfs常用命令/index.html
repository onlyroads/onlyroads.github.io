    hdfs dfsadmin -safemode 安全模式
  • 查看、进入、离开安全模式：hdfs dfs admin -safemode get/enter/leave
    [hadoop@hadoop100 bin]$ hdfs dfsadmin -safemode
 Usage: hdfs dfsadmin [-safemode enter | leave | get   | wait]
[hadoop@hadoop100 bin]$ hdfs dfsadmin -safemode get
 Safe mode is OFF
  • OFF离开安全模式，客户端可以对hdfs集群进行读写
  • ON进入安全模式，客户端对hdfs集群可以读，但是不能写入
    不可以写
    [hadoop@hadoop100 bin]$ hdfs dfsadmin -safemode enter
Safe mode is ON 
[hadoop@hadoop100 bin]$ hadoop fs -put yarn /
put: Cannot create file/yarn._COPYING_. Name node is in safe mode.
    可以查看
    [hadoop@hadoop100 bin]$ hadoop fs -ls /
-rwxr-xr-x   1 hadoop supergroup          4 2020-11-26 17:07 /a.log
-rw-r--r--   1 hadoop supergroup         11 2020-11-22 14:29 /input
drwxr-xr-x   - hadoop supergroup          0 2020-11-22 14:30 /output
  • 集群进入安全模式可能有以下情况
    丢失块，达不到阈值集群自动会进入安全模式
    业务需要进行维护，手动进入安全模式
    hdfs集群重新启动刚开始会进入安全模式

    是指每个节点磁盘使用率-平均磁盘使用率<10%
    第一个节点使用率是 90%
    第二个节点使用率是 60%
    第一个节点使用率是 80%
    平均使用率就是 230%/3=76%
    第一个节点 90% -76% = 14% 多了4%
    第二个节点 60% -76% = -16% 少-16%
    第三个节点 80% -76%= 4% 满足

    hdfs diskbalancer 单个节点多块磁盘平衡

  • 配置
    假设有data01，data02，data03三个磁盘
    在hdfs-site.xml中添加如下信息
    <property>
      <name>dfs.datanode.data.dir </name>
      <value>/data01/dfs/dn,/data02/dfs/dn,/data03/dfs/dn</value>
</property>
<property>
      <name>dfs.disk.balancer.enabled</name>
      <value>true</value>
</property>
  • 命令查看
    [hadoop@hadoop100 hadoop]$ hdfs diskbalancer
usage: hdfs diskbalancer [command] [options]
DiskBalancer distributes data evenly between different disks on a
datanode. DiskBalancer operates by generating a plan, that tells datanode
how to move data between disks. Users can execute a plan by submitting it
to the datanode.
To get specific help on a particular command please run
hdfs diskbalancer -help <command>.
  --help <arg>   valid commands are plan | execute | query | cancel |
                 report
  • 版本支持
    Apache hadoop2.x 不支持该命令
    Apache hadoop3.x 支持该命令
    CDH hadoop2.x 支持该命令
  • 执行命令
    hdfs diskbalancer -plan 主机名
  • 生产中写脚本每晚在业务低估去执行该命令
hdfs hadoop
