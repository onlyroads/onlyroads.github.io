内存

[root@hadoop100 ~]# free -m
              total        used        free      shared  buff/cache   available
Mem:           3934         376        3171          12         385        3267
Swap:          3967           0        3967

预留内存最好在15%
376 / 3943 = 8%
swap 因为内存不够，使用部分磁盘空间来充当内存使用，虽然可以解决内存紧缺的问题，但是效率不高；尤其是大数据，swap哪怕设置了大小，也尽量设置惰性使用

杀死进程
ps -ef | grep 名称
有可能匹配多个，仔细确认进程是否是自己想要杀的进程。
kill -9 pid 【高危命令】其他高危命令有：rm -rf，echo “” > file
kill -9 111 112 113 三个进程一杀，会造成生成上的误杀事故
全局杀
kill -9 $(pgrep -f 匹配字符)

端口号

netstat -nlp | grep pid/名称

[root@hadoop100 ~]# netstat -nlp|grep httpd
tcp6       0      0 :::80                   :::*                    LISTEN      2261/httpd

:::80、0.0.0.0: 80、192.168.1.100:80
80端口号可以对外，window或者其他服务器可以ip + 80进行访问
localhost、127.0.0.1代表本机
80端口仅本机额其他服务可以访问，window或者其他服务器无法访问

可以在/etc/httpd/conf/httpd.conf文件中修改
[root@hadoop100 conf]# pwd
/etc/httpd/conf
[root@hadoop100 conf]# vim httpd.conf
#Listen 12.34.56.78:80
Listen 80

总结：
a. 有进程PID不一定就有端口号
b. 服务的通信交流，其实就是要 ip + 端口号

[root@hadoop100 ~]# telnet 192.169.1.100 221
Trying 192.169.1.100...
telnet: connect to address 192.169.1.100: Connection refused

Connection refused 【错误三】 command not found【错误一】 permission denied【错误二】
linux机器去访问 服务器的服务
先 ping ip
再 telnet ip port

[root@hadoop100 ~]# telnet hadoop100 22
Trying 192.168.1.100...
Connected to hadoop100.
Escape character is '^]'.
SSH-2.0-OpenSSH_7.4

windows电脑去访问 服务器的服务
先 ping ip
再 telnet ip port
注意
今后如果ping ip可以通，但是telnet不通，但是ip和端口号都开了
很有可能是端口号对应的是localhost/127.0.0.1，外网访问不了

commond not fount

原因：
没有安装/没有配置环境变量
配置环境变量：
export JAVA_HOME=/usr/java/jdk1.8
export PATH=$JAVA_HOME/bin:$PATH 推荐/bin目录应该配置在PATH前面
这样当系统查找命令时，会先从配置的环境变量中查找，一旦找到PATH后面的内容就不再进行查找了
